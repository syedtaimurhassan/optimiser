MathOpt - solver backend sample

Problem: solve a tiny linear model and compare how the MathOpt frontend sends it to different solver backends.

Show code 
import { initMathOpt, MathOpt, setWorkerBridgeEnabled } from 'or-tools-wasm/mathopt';

setWorkerBridgeEnabled(true);
await initMathOpt();

const model = MathOpt.Model('basic');
const x = model.addVariable({ lowerBound: 0, upperBound: 1, name: 'x' });
const y = model.addVariable({ lowerBound: 0, upperBound: 1, name: 'y' });

model.addLinearConstraint({ upperBound: 1, terms: [
  { variable: x, coefficient: 1 },
  { variable: y, coefficient: 1 },
]});
model.maximize([
  { variable: x, coefficient: 2 },
  { variable: y, coefficient: 1 },
]);

const result = await MathOpt.solve(model, {
  solverType: MathOpt.SolverType.GLOP,
  timeLimitSeconds: 5,
  glop: new MathOpt.GlopParameters({
    usePreprocessing: true,
    useScaling: true,
  }),
});
console.log(result.objectiveValue, result.variableValues);

Adapted from ortools/math_opt/samples/python/basic_example.py. The browser demo solves the same two-variable linear model with a selectable MathOpt backend. GLPK runs with a single thread.

Model

  • The model has two variables, one capacity constraint, and one linear objective.
  • The backend selector sends the same MathOpt model to different solver backends.
  • Integer-capable backends use binary variables; LP backends use continuous variables between 0 and 1.
  • The solution table shows how the selected backend values the same small model.
